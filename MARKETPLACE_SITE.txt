VScode Mobile — Plugin Marketplace Site Plan
============================================

STACK
-----
- Framework: Next.js 14 (App Router)
- Hosting: Vercel
- Database: Vercel Postgres (or PlanetScale)
- Storage: Vercel Blob (for plugin .js files)
- Auth: GitHub OAuth (for submitters)

PAGES
-----
/ — Homepage, featured plugins, search bar
/plugins — Full plugin list with filters (category, language, rating)
/plugins/[id] — Plugin detail page (readme, version, install URL, author)
/submit — Plugin submission form
/developer — Developer docs + API reference
/admin — Private review dashboard (password protected)

PLUGIN SUBMISSION FLOW
----------------------
1. Developer fills out /submit form:
   - Plugin name, description, category
   - GitHub repo URL (optional)
   - Upload plugin .js file (max 500KB)
   - Contact email
2. Form submits to /api/submit → stores in DB with status "pending"
3. Owner (you) gets email notification
4. You review at /admin:
   - Read plugin source code inline
   - Approve → plugin goes live, .js stored in Vercel Blob
   - Reject → email sent to developer with reason
5. Approved plugin appears in marketplace with install URL:
   https://vscode-mobile-plugins.vercel.app/p/[plugin-id]/plugin.js

MARKETPLACE API (used by the app)
----------------------------------
GET /api/plugins              — list all approved plugins (JSON)
GET /api/plugins?q=search     — search plugins
GET /api/plugins?category=ai  — filter by category
GET /api/plugins/[id]         — single plugin metadata
GET /p/[id]/plugin.js         — serve plugin JS file (with CORS *)

DEVELOPER TESTING
-----------------
Developers can test plugins locally before submitting:
1. Enable Developer Mode in app Settings
2. Go to Settings → Developer → Load Local Plugin
3. Paste plugin JS code directly into the text area
4. Plugin runs immediately in the editor sandbox
