VSCode Android — Plugin API Reference

Build powerful plugins for VSCode Android using the JavaScript Plugin API.

Quick Start

A plugin is a single JavaScript file that calls VscodePlugin.register(). It runs inside the Monaco Editor WebView.

VscodePlugin.register({
  id: 'my-plugin',
  name: 'My Plugin',
  version: '1.0.0',
  description: 'Does something awesome',
  author: 'Your Name',

  activate(ctx) {
    // Plugin is ready — ctx gives you access to all APIs
    ctx.ui.showMessage('Hello from My Plugin!');
  },

  deactivate() {
    // Cleanup when plugin is removed
  }
});

VscodePlugin.register(manifest)

Registers your plugin with the runtime. Must be called at the top level of your plugin file.

FieldTypeDescription
idstringUnique plugin identifier (kebab-case)
namestringDisplay name shown in marketplace
versionstringSemantic version (e.g. 1.0.0)
descriptionstringShort description
authorstringAuthor name or GitHub handle
activate(ctx)functionCalled when plugin loads — main entry point
deactivate()function optionalCalled when plugin is removed

ctx.editor — Editor API

Interact with the Monaco Editor instance.

ctx.editor.getText()

Returns the full content of the current file as a string.

const content = ctx.editor.getText();
console.log(content);

ctx.editor.setText(text)

Replaces the entire editor content.

ctx.editor.setText('// New content\nconsole.log("hello")');

ctx.editor.getSelection()

Returns the currently selected text, or empty string if nothing is selected.

const selected = ctx.editor.getSelection();

ctx.editor.insertText(text)

Inserts text at the current cursor position, replacing any selection.

ctx.editor.insertText('// inserted comment\n');

ctx.editor.getCursorPosition()

Returns { line, column } of the current cursor.

const pos = ctx.editor.getCursorPosition();
console.log(`Line: ${pos.line}, Col: ${pos.column}`);

ctx.editor.getLanguage()

Returns the current language ID (e.g. "javascript", "python", "dart").

const lang = ctx.editor.getLanguage();

ctx.editor.setLanguage(languageId)

Changes the syntax highlighting language.

ctx.editor.setLanguage('typescript');

ctx.editor.formatDocument()

Triggers Monaco's built-in document formatter.

ctx.editor.formatDocument();

ctx.editor.addAction(action)

Adds a custom command to the editor's command palette.

ctx.editor.addAction({
  id: 'my-plugin.hello',
  label: 'My Plugin: Say Hello',
  keybindings: [], // optional Monaco keybinding codes
  run() {
    ctx.ui.showMessage('Hello!');
  }
});

ctx.ui — UI API

Show messages and interact with the app UI.

ctx.ui.showMessage(text)

Shows a snackbar notification at the bottom of the screen.

ctx.ui.showMessage('File saved!');

ctx.ui.showError(text)

Shows an error snackbar (red).

ctx.ui.showError('Something went wrong');

ctx.ui.showInputBox(options)

Shows a dialog with a text input. Returns a Promise resolving to the entered string, or null if cancelled.

const name = await ctx.ui.showInputBox({
  title: 'Enter name',
  placeholder: 'e.g. MyClass',
  value: ''
});
if (name) ctx.editor.insertText(name);

ctx.ui.showQuickPick(items)

Shows a selection dialog. Returns a Promise resolving to the selected item string, or null.

const choice = await ctx.ui.showQuickPick([
  'Option A', 'Option B', 'Option C'
]);
if (choice) ctx.ui.showMessage(`You picked: ${choice}`);

ctx.hooks — Lifecycle Hooks

React to editor events.

ctx.hooks.onSave(callback)

Called whenever the user saves the current file.

ctx.hooks.onSave((filePath) => {
  ctx.ui.showMessage(`Saved: ${filePath}`);
});

ctx.hooks.onFileOpen(callback)

Called when a new file is opened in the editor.

ctx.hooks.onFileOpen((filePath) => {
  console.log('Opened:', filePath);
});

ctx.hooks.onChange(callback)

Called on every content change. Use sparingly — fires on every keystroke.

ctx.hooks.onChange((content) => {
  // content = full editor text after change
});

ctx.hooks.onCursorMove(callback)

Called when the cursor position changes.

ctx.hooks.onCursorMove(({ line, column }) => {
  console.log(line, column);
});

ctx.storage — Persistent Storage

Key-value storage scoped to your plugin. Data persists across app restarts.

ctx.storage.get(key)

Returns a Promise resolving to the stored value, or null if not set.

const token = await ctx.storage.get('api_token');

ctx.storage.set(key, value)

Stores a string value. Returns a Promise.

await ctx.storage.set('api_token', 'sk-...');

ctx.storage.remove(key)

Removes a stored key. Returns a Promise.

await ctx.storage.remove('api_token');

ctx.http — HTTP Requests

Make HTTP requests from your plugin. Requests are routed through Tor if the user has it enabled.

ctx.http.get(url, headers?)

Performs a GET request. Returns a Promise resolving to { status, body }.

const res = await ctx.http.get('https://api.example.com/data');
if (res.status === 200) {
  const data = JSON.parse(res.body);
}

ctx.http.post(url, body, headers?)

Performs a POST request with a JSON body. Returns a Promise resolving to { status, body }.

const res = await ctx.http.post(
  'https://api.example.com/submit',
  { key: 'value' },
  { 'Authorization': 'Bearer token' }
);
Note: HTTP requests from plugins respect the user's Tor proxy setting. When Tor is enabled, all requests are routed through SOCKS5 on 127.0.0.1:9050 via Orbot.

Full Examples

Word Counter Plugin

VscodePlugin.register({
  id: 'word-counter',
  name: 'Word Counter',
  version: '1.0.0',
  description: 'Shows word count in status bar',
  author: 'example',

  activate(ctx) {
    const count = () => {
      const text = ctx.editor.getText();
      const words = text.trim().split(/\s+/).filter(Boolean).length;
      ctx.ui.showMessage(`Words: ${words}`);
    };

    ctx.editor.addAction({
      id: 'word-counter.count',
      label: 'Word Counter: Count Words',
      run: count
    });

    ctx.hooks.onSave(() => count());
  }
});

Auto Header Plugin

VscodePlugin.register({
  id: 'auto-header',
  name: 'Auto Header',
  version: '1.0.0',
  description: 'Inserts a file header comment',
  author: 'example',

  activate(ctx) {
    ctx.editor.addAction({
      id: 'auto-header.insert',
      label: 'Auto Header: Insert Header',
      async run() {
        const author = await ctx.storage.get('author') || 'Unknown';
        const date = new Date().toISOString().split('T')[0];
        const lang = ctx.editor.getLanguage();
        const comment = lang === 'python'
          ? `# Author: ${author}\n# Date: ${date}\n\n`
          : `// Author: ${author}\n// Date: ${date}\n\n`;
        const current = ctx.editor.getText();
        ctx.editor.setText(comment + current);
      }
    });
  }
});

Publishing to Marketplace

To publish your plugin to the VSCode Android Marketplace:

  1. Host your plugin JS file at a public URL (GitHub raw, CDN, etc.)
  2. Go to vscode-mobile-plugins.vercel.app
  3. Click Submit Plugin and fill in the form
  4. Provide: name, description, author, category, and the raw JS URL
  5. The maintainer reviews and approves your submission
  6. Once approved, your plugin appears in the in-app Marketplace
Developer Mode: To test your plugin before publishing, enable Developer Mode in Settings. You can then paste your plugin JS directly into the local plugin loader and run it instantly without publishing.
